# 语音识别与合成计划

## 一、架构总览

```
┌─ 输入（ASR）──────────────────────────────────────────┐
│                                                        │
│  用户发语音 record 段                                    │
│    → voice.js: 抽 URL → 下载 → ffmpeg 转 WAV            │
│    → base64 → 送 mimo-v2-omni 做 chat completions       │
│      （[input_audio] 格式，同 vision 的 image_url）      │
│    → 取回复 → 替换 userText → AI 看到转写文字            │
│                                                        │
├─ 输出（TTS）────────────────────────────────────────────┤
│                                                        │
│  触发（命令 / 随机 / 引用朗读）                           │
│    → 查当前人格的 voice 配置（voice_id + style）          │
│    → LLM 输出回复时可选附加【语音风格：xxx】标签            │
│    → tts.js: 组合 style + text + voice                  │
│    → POST /v1/chat/completions (audio)                  │
│    → base64 WAV → 临时文件 → QQ 语音消息                  │
│    → 同时发文字版                                        │
│                                                        │
├─ 人格-音色映射──────────────────────────────────────────┤
│                                                        │
│  SKILL.椿.md  frontmatter:                              │
│    voice: { voice_id: 茉莉, style: 慵懒危险 }            │
│  SKILL.长离.md frontmatter:                             │
│    voice: { voice_id: 冰糖, style: 温柔知性 }            │
│                                                        │
│  前端可上传音频样本 → voiceclone → 绑定到人格             │
│                                                        │
└────────────────────────────────────────────────────────┘
```

## 二、语音识别（ASR）—— `lib/voice.js`

### API 确认

查 MiMo 官方文档，`input_audio` 格式如下（`mimo-v2.5` 和 `mimo-v2-omni` 均支持）：

```json
{
  "model": "mimo-v2.5",
  "messages": [{
    "role": "user",
    "content": [
      { "type": "input_audio", "input_audio": { "data": "https://..." } },
      { "type": "text", "text": "请描述这段音频" }
    ]
  }]
}
```

**关键点：**
- `input_audio.data` 接受**直接 URL** 或 **data URI**（`data:audio/wav;base64,...`），同 vision 的 `image_url.url` 模式
- 支持超长音频（官方称 >10 小时连续理解），QQ 语音消息（通常 <60s）无限制
- `mimo-v2.5`（本项目的 `vision: false`）支持音频理解但不支持图片——说明 vision 标签独立于音频理解
- 如果 NapCat 语音 URL 可外网访问（如 QQ 的 CDN 直链），可直接传 URL，省去下载+转码

### 核心思路

跟 vision 传图片一样——语音文件 URL 或 base64 塞进 messages 的 `input_audio` 块，送 MiMo 多模态模型做 chat completions，模型直接返回转写文字。

```
record 语音
  → extractVoiceUrl(session) → 取 NapCat 语音 URL

  方案 A（默认）：下载 → base64（可靠，不受 URL 有效期影响）
  → downloadVoiceFile(url) → 临时文件 .silk/.amr/.mp3
  → convertToWav(file)     → ffmpeg → 16kHz 16bit mono WAV
  → readFileSync → base64 data URI

  方案 B（可选）：直接传 URL（省下载时间，但 URL 可能过期）
  → 直接取 URL 原文

  → requestChatCompletions([
      { role: 'user', content: [
        { type: 'text', text: '请将这段语音转写成文字，只返回文字本身。不要添加其他内容。' },
        { type: 'input_audio', input_audio: { data: '<URL 或 data URI>' } }
      ]}
    ], { model: 'mimo-v2.5' 或 'mimo-v2-omni', provider: 'mimorium', apiKey })
  → 取回复文本 → 清理临时文件
  → 注入 userText: "[语音转文字：xxx]"
```

### 新增文件：`lib/voice.js`

| 函数 | 职责 |
|------|------|
| `extractVoicePayload(session)` | 从 `session.event.message` 抽 `type === 'record'` 的 `{ url, file }`；CQ-string 回退从 `content` 提取 |
| `transcribeVoice(voicePayload, config)` | 顶层入口：下载 → 转码 → 调模型 → 返回 `{ text }` |
| `downloadVoiceFile(url, destPath)` | HTTP GET 下载语音文件到临时路径，限 2MB / 15s |
| `convertToWav(srcPath)` | `ffmpeg -y -i "{src}" -ar 16000 -ac 1 -sample_fmt s16 "{src}.wav"`；ffmpeg 不存在则尝试 `@xqq/silk` 解码 |
| `callModelAsr(wavPath, modelConfig)` | 读 WAV → base64 → 构造 `input_audio` messages → `requestChatCompletions` → 取回复文字 |

### 辅助改动的文件

| 文件 | 改动 |
|------|------|
| `utils.js` | 新增 `extractVoiceUrls(content)` — 匹配 `[CQ:record,...url=...]` 和 `<record url=.../>` |
| `api.js` | 新增 `extractRecordUrlFromElements(session)` — 结构化消息中抽 record 的 url/file；新增 `downloadVoiceFile(url, destPath)` |
| `message-reader.js` | `collectSegmentFeatures`：record 不再设 `hasVisual`，改为新 feature `hasAudio`；`createFeatureState` 加 `hasAudio: false`；`extractSegmentText`：record 分支检查 `data._transcribedText` → `[语音转文字：xxx]`；`analyzeIncomingMessage` 返回加 `hasAudio` |
| `conversation.js` | `getQuoteContentText`：`s.type === 'record'` 时检查 `s.data?._transcribedText` |
| `index.js` | `require('./voice')`；在 `userText` 构造后检测 `hasAudio` → 调 `transcribeVoice` → 成功则 `userText = "[语音转文字：${text}]"`；设 15s 超时，失败不阻塞 |

### 容错处理

- 识别失败/超时 → `session.send("没听清，能打字吗？")`，不阻塞主流程
- 语音里无人声/只有噪音 → 全模态 LLM 能理解（"背景噪音，没有语音内容"），返回"没有听清"即可
- 超时设 10s（MiMo 音频理解响应比图片快，无需 15s）

### 替代 ASR 方案（可选）

如果以后想切到专用 ASR API（百度/腾讯语音识别），`voice.js` 加一个 `asrProvider` 配置：
```javascript
asrProvider: 'mimo'       // 当前默认：mimo
asrProvider: 'baidu'      // 可选：百度短语音识别
asrProvider: 'tencent'    // 可选：腾讯语音识别
```
前端的 Agent Console 人格设置里可以切换。当前阶段只实现 `mimo`，保留接口和配置字段即可。

### 依赖

- `silk-wasm`（已安装 v3.7.1）— ffmpeg 不可用时的备用 Silk → PCM 解码（`decode(input, sampleRate)`），再通过 ffmpeg 或无操作转为 WAV
- `ffmpeg` 服务端已有（`/usr/bin/ffmpeg`），silk-wasm 仅作 fallback

### 注意事项

- `voice.js` 不依赖 `vision.js`，但复用 `api.js` 的 `requestChatCompletions`
- 不修改现有 vision 流程
- 转录调用独立于主 chat 流程：先转录 → 替换 userText → 主 chat 继续

## 三、语音合成（TTS）—— `lib/tts.js`

### 新增文件：`lib/tts.js`

| 函数 | 职责 |
|------|------|
| `synthesizeSpeech(text, config, options)` | 调 MiMo `/v1/chat/completions`（audio 模式）→ base64 WAV → 返回 Buffer |
| `sendVoiceMessage(session, audioBuf)` | 写临时 WAV 文件 → `session.send({ type: 'record', data: { file } })` → 延迟删除 |
| `generateRandomSpeech(session, ctx)` | 调 AI 生成一句话 → synthesizeSpeech → sendVoiceMessage |
| `getBuiltinVoices()` | 返回预设音色列表 |

`synthesizeSpeech` 内部逻辑：
```
1. 构造 messages: [{ role: 'user', content: 风格描述 }, { role: 'assistant', content: text }]
2. 构造 audio: { format: 'wav', voice: options.voice || '冰糖' }
3. POST /v1/chat/completions → 取 choices[0].message.audio.data
4. Buffer.from(data, 'base64') → 返回 Buffer
```

### MiMo TTS API — 实际格式（走 chat/completions，不是 audio/speech）

```
POST https://token-plan-cn.xiaomimimo.com/v1/chat/completions
api-key: {miMoKey}
Content-Type: application/json

{
  "model": "mimo-v2.5-tts",              // 或 mimo-v2.5-tts-voicedesign
  "messages": [
    {
      "role": "user",
      "content": "活泼可爱、带点撒娇的语气，语速偏快"
    },
    {
      "role": "assistant",               // 待合成文本必须放 assistant
      "content": "要合成语音的文字内容"
    }
  ],
  "audio": {
    "format": "wav",
    "voice": "冰糖"                       // 音色 ID
  }
}

→ Response:
{
  "choices": [{
    "message": {
      "audio": {
        "data": "base64_encoded_wav..."
      }
    }
  }]
}
```

**关键规则：**
- 文本必须放 **assistant** role，不能放 user
- **user** role 是可选的风格描述（不会被朗读）
- 输出是 24kHz PCM16LE mono WAV，base64 编码在 `choices[0].message.audio.data`
- Authorization 用 MiMo 聊天 API 同样的 key（`ai-mimorium-key.txt`）
- NapCat 收到 WAV 文件会自动转 Silk 发送

### 三种音色控制模式

| 模式 | 模型 | 控制方式 |
|------|------|----------|
| **预设音色** | `mimo-v2.5-tts` | `audio.voice` 指定音色 ID（见下表） |
| **文本描述音色** | `mimo-v2.5-tts-voicedesign` | `user` 消息描述想要的嗓音（不需要 `voice` 参数） |
| **音色克隆** | `mimo-v2.5-tts-voiceclone` | `audio.voice` 传 base64 音频样本 |

| 冰糖 | `冰糖` | 中文 | 女 |
| 茉莉 | `茉莉` | 中文 | 女 |

### 风格控制（`user` 消息）

user 消息里写自然语言即可控制说话的风格、情绪、语气，例如：

| 效果 | user 内容示例 |
|------|-------------|
| 开心活泼 | `"开心活泼，语调上扬"` |
| 慵懒疲惫 | `"慵懒、刚睡醒、略带沙哑"` |
| 东北方言 | `"东北话，豪爽的语气"` |
| 夹子音撒娇 | `"夹子音，撒娇的感觉"` |
| 角色扮演 | `"模仿孙悟空，用猴哥的语气说话"` |
| 导演模式 | `"【角色】温柔御姐，30岁知性女性\n【场景】哄孩子睡觉\n【指导】语速缓慢，轻声细语"` |
| 复合控制 | `"可爱又撒娇、软软的夹子音，语速要慢"` |

辅助使用 **Audio Tag** 在文本中嵌入：

```
（紧张，深呼吸）呼……冷静，冷静。不就是一个面试吗……
（极其疲惫，有气无力）师傅……到地方了叫我一声……（长叹一口气）
（提高音量喊话）大姐！这鱼新鲜着呢！
```

支持的 tag：吸气、深呼吸、叹气、紧张、颤抖、笑、撒娇、激动 等。

### 触发方式

#### A. 命令触发（handler.js 新增）

| 命令 | 行为 |
|------|------|
| `东雪莲说句话` | AI 结合最近聊天上下文生成一句话 → TTS → 发送语音 + 文字原文 |
| `东雪莲朗读 xxx` | 将 `xxx` 直接 TTS → 发送语音 |
| `朗读`（引用消息时） | TTS 被引用的消息内容 |

```javascript
// handler.js 追加
if (/^东雪莲说句话$/.test(plain)) {
  const recentHistory = getConversationHistory(session)
  const prompt = `根据最近聊天内容，用东雪莲的语气说一句简短的话（20字以内），不要问句，直接说。`
  const text = await callOpenAI([{ role: 'system', content: prompt }], false, { max_tokens: 50 })
  if (text) {
    const audioBuf = await synthesizeSpeech(text, config, { voice: '冰糖' })
    await sendVoiceMessage(session, audioBuf)
    await session.send(text)  // 同时发文字版
  }
  return handled()
}

if (plain.startsWith('东雪莲朗读')) {
  const text = plain.slice(5).trim()
  if (text) {
    const audioBuf = await synthesizeSpeech(text, config, { voice: '冰糖' })
    await sendVoiceMessage(session, audioBuf)
  }
  return handled()
}
```

#### B. 低概率随机触发（index.js 集成）

在随机回复判断处（`randomTriggered` 逻辑旁）增加一个低概率语音触发：

```javascript
// 约 0.5%~1% 的概率触发语音回复而非文字
const ttsRandomTriggered = randomTriggered && Math.random() < 0.05  // 5% 的随机回复走语音
```

触发后：
1. `chat()` 正常走 AI 生成文字回复
2. 将文字回复送入 `synthesizeSpeech` → 发语音
3. 同时保留文字回复（或文字+语音一起发）

随机 TTS 冷却：同一条 channel 每 5 分钟最多一次语音随机触发。

### 音频文件处理

```
TTS API 返回 base64 WAV Buffer（24kHz PCM16LE mono）
  → 写入临时文件 /tmp/tts-{uuid}.wav
  → session.send([{ type: 'record', data: { file: '/tmp/tts-{uuid}.wav' } }])
  → 发送完成后删除临时文件（setTimeout + 5s 延迟以保证发送完成）
```

NapCat 收到 `record` 消息请求后，会自动将 WAV 转为 QQ 协议所需的 Silk 格式再发送，用户端收到的是普通语音消息。

注意：不要用 `response_format: "mp3"`，因为 MiMo TTS 是通过 chat/completions + audio 参数调用的，只支持 `wav` 和 `pcm16` 两种格式。

## 六、风险控制

| 风险 | 应对 |
|------|------|
| ASR API 超时/失败 | Promise.race 设 15s 超时 → 回退 `【语音消息】`，不阻塞聊天 |
| TTS API 失败 | 发文字回复代替，不抛异常 |
| 语音文件超过 2MB | downloadVoiceFile 限制 content-length ≤ 2MB |
| ffmpeg 不可用 | try → catch → 回退（语音识别跳过，语音合成无法进行则发文字） |
| 随机语音太频繁 | channel 级 5 分钟冷却，频率 ≤ 5% 的随机回复 |
| NapCat 不支持自动转码 | 备选方案：装 `silk-v3-encoder` 手动转 Silk 再发送 |
| API key 泄露 | key 从文件读取，不硬编码；计划文档不包含实际 key |
| 1.6GB 内存耗尽 | 临时文件用完即删；ASR 全程流式不保留大 buffer 在内存 |

## 八、人格-音色映射系统

### 设计目标

每个自定义人格（SKILL.\<name\>.md）可以绑定独立的音色+说话风格，TTS 时自动使用当前人格的 voice 和 style，实现"换人格 = 换声音"。

### Persona Frontmatter 扩展

在现有 `SKILL.椿.md`、`SKILL.长离.md` 等文件的 YAML frontmatter 中增加 voice 字段：

```yaml
---
name: 长离
description: 长离，《鸣潮》今州华胥研究院「执礼」。
will: 1.3
lore: wuwa-lore
voice:
  provider: mimo                # 默认 mimo
  model: mimo-v2.5-tts          # 或 voicedesign / voiceclone
  voice_id: 冰糖                # 预设音色 ID（预设模式用）
  style: 温柔知性、御姐音、语速沉稳  # 默认说话风格
  sample_file: null             # 克隆音色的音频文件路径（前端上传后填写）
  description: 温婉知性的成年女性声音  # 给 LLM 参考的描述
---
```

**不同人格的预设示例：**

| 人格 | voice_id | style | 说明 |
|------|----------|-------|------|
| 东雪莲（默认） | `冰糖` | 活泼直率、带点不耐烦 | 默认东雪莲语气 |
| 长离 | `冰糖` | 温柔知性、御姐音、语速沉稳 | 同一音色但风格不同 |
| 椿 | `茉莉` | 慵懒危险、语气飘忽、带笑意 | 更适合的角色音色 |
| 特蕾西娅 | `茉莉` | 温柔悲悯、语调平和、略带忧伤 | |
| 爱弥斯 | `冰糖` | 活泼元气、少女感、语速较快 | |

> 预设音色表见上（冰糖 = 中文女声，茉莉 = 中文女声，苏打 = 中文男声，白桦 = 中文男声）

### 读取逻辑（`tts.js` 新函数）

```javascript
function resolvePersonaVoice(personaName) {
  // 1. 读 persona 文件 frontmatter → voice 字段
  // 2. 没有 voice 配置 → 返回默认 { voice_id: '冰糖', style: '' }
  // 3. 有 sample_file → 标记为克隆模式
  // 4. 有 description 且无 style → 用 description 当 style
  return { model, voice_id, style, sample_file }
}
```

### 默认人格（东雪莲）的音色

默认人格（无自定义 persona 时用的东雪莲模式）也需要绑定音色。在 `runtime-config.js` 中增加默认 voice 配置：

```javascript
// runtime-config.js 新增
defaultVoice: {
  model: 'mimo-v2.5-tts',      // 上传克隆音频后改为 'mimo-v2.5-tts-voiceclone'
  voice_id: '冰糖',              // 预设音色（克隆后改为 'custom:dongxuelian'）
  style: '活泼直率、带点不耐烦',
  sample_file: null,             // 上传克隆音频后填写路径
}
```

### 音色克隆的 TTS 调用方式

MiMo voiceclone API 每次请求都需要传完整音频 base64（上限 10MB）。读取流程：

```
TTS 调用时检测到 model === 'mimo-v2.5-tts-voiceclone'
  → 从 frontmatter 或 config 读 sample_file 路径
  → fs.readFileSync → base64
  → 构造 data URI: "data:audio/{ext};base64,{base64}"
  → 设置 audio.voice = data URI
  → POST /v1/chat/completions
```

音频样本较小（30秒 MP3 ≈ 480KB），每次实时读取即可，无需持久缓存 base64。

### 数据存储

- Persona voice 配置存在 `SKILL.<name>.md` 的 frontmatter 中
- 默认人格（东雪莲）的 voice 配置存在 `runtime-config.js` 中
- 克隆音频样本文件存在 `data/ai-voices/{personaName}.{ext}`
- `persona.js` 加 `parsePersonaVoice(frontmatter)` 解析函数

### 实现改动

| 文件 | 改动 |

## 九、LLM 自动语气切换（群聊动态匹配语气）

### 回答你的问题

> "意思是让ai说话还要给他一个自然语言描述对吗，让他用不同的语气"
> "群聊该如何自动切换用不同的语气呢，靠chat的llm来帮忙吗"

**是的，完全靠 LLM 自己来判断。** 不写 if-else 规则映射关键词到风格，而是把风格选择权交给大模型。

### 实现方式：System Prompt 注入 + 后处理解析

#### Step 1: 在系统提示末尾注入语音指令

```javascript
// chat.js 构造 systemPrompt 末尾追加
const voiceInstruction = personaVoice
  ? `当前人格音色：${personaVoice.voice_id}（${personaVoice.style || personaVoice.description || ''}）`
  : ''

// 仅当 TTS 模式激活时才注入此指令
const ttsSystemNote = `如果需要使用语音回复，请在回复的末尾附加语音风格标签，格式如下：
【语音风格：<风格描述>】
例如回复：
今天天气真好啊
【语音风格：开心活泼】

风格描述要用自然语言，控制在 20 字以内，说明语气、情绪、语速等。
当前音色是${personaVoice?.voice_id || '冰糖'}，请基于这个音色选择适合的语气。`
```

#### Step 2: LLM 生成回复 → 提取风格标签

```javascript
// chat.js 或 chatResult handler
function extractVoiceStyle(replyText) {
  const match = replyText.match(/【语音风格：(.+?)】/)
  if (match) {
    const style = match[1].trim()
    const cleanText = replyText.replace(/【语音风格：.+?】/, '').trim()
    return { text: cleanText, style }
  }
  return { text: replyText, style: null }  // 没有风格标签 → 用默认 style
}
```

#### Step 3: 使用风格标签生成 TTS

```javascript
async function replyWithVoice(session, replyText, config, personaVoice) {
  const { text, style } = extractVoiceStyle(replyText)
  const voiceStyle = style || personaVoice.style || ''
  const audioBuf = await synthesizeSpeech(text, config, {
    voice: personaVoice.voice_id || '冰糖',
    style: voiceStyle,
    model: personaVoice.model,
  })
  await sendVoiceMessage(session, audioBuf)
}
```

### 示例流程

```
群友: "今天好累啊"
→ LLM 系统提示: "如果需要语音回复，在末尾加【语音风格：xxx】"
→ LLM 回复:
  "累了就早点休息吧，别熬夜了"
  【语音风格：温柔关心、语气轻缓】
→ tts.js 提取 style="温柔关心、语气轻缓" + voice="冰糖"
→ POST /v1/chat/completions (audio)
  user: "温柔关心、语气轻缓"
  assistant: "累了就早点休息吧，别熬夜了"
→ 发送语音消息 + 文字
```

### 关键设计

- **风格标签不强制**：LLM 可以选择不加，此时用人格的默认 style
- **TTS prompt 只在可能触发语音的场景注入**（命令触发、随机触发），不影响纯文字聊天
- **LLM 自行判断语境**：群友在生气时，LLM 自然会输出「安抚、耐心」风格；在开玩笑时输出「活泼、调侃」风格

### 上传流程细节

```
浏览器用户选择音频文件（≤30秒，≤10MB）
  → 推荐格式：MP3（最通用，压缩率高）或 WAV（原始质量）
  → 浏览器还可读取：OGG、M4A/AAC、FLAC、WebM（但 MiMo voiceclone 只保证支持 MP3/WAV，其他格式服务端需 ffmpeg 转码）
  → 前端文件选择器设 `<input accept=".mp3,.wav,.ogg,.m4a,.flac,.webm,audio/*">`
  → FileReader.readAsDataURL(file) → 得到 data URI（含 base64）
  → 提取纯 base64 部分
  → POST /dashboard/api/agent/tts/clone
     body: { personaName, audioBase64, mimeType: 'audio/mpeg' }
  → 后端：
     1. 验证 base64 解码后 ≤ 10MB
     2. 保存到 data/ai-voices/{personaName}.{ext}
     3. 调 MiMo voiceclone 做一次合成验证（synthesizeSpeech("测试语音", { model: 'mimo-v2.5-tts-voiceclone', voice: dataUri })）
     4. 验证通过 → 更新 persona frontmatter 或 runtime-config 的 voice 段
        model: mimo-v2.5-tts-voiceclone
        sample_file: voices/{personaName}.mp3
     5. 返回 { ok: true, message: '音色克隆成功' }
```
