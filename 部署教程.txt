LianLianBot 传统 Linux 部署教程
===============================

说明：这份文档是高级/传统 Linux 服务器部署备份，适合需要手工排查或维护旧服务器的人。普通 Windows 用户优先看 README 的 Windows EXE 部署器；已有 Dashboard 的维护者优先使用 Dashboard 部署页。

准备工作
--------
1. 一台 Linux 服务器（Ubuntu 22.04 推荐），能 ssh 连接
2. 一个 QQ 号（用于 Bot 登录）
3. API key：任意已支持供应商均可选填；不填也能完成基础部署，但 AI 回复不可用


方式一：setup.sh 一键部署
--------------------------

1. 获取 setup.sh（二选一）

   a) 直接下载到服务器（推荐）
      wget https://raw.githubusercontent.com/qiongtu2077/dongxuelian-qqbot/main/setup.sh

   b) 或者先在本地 git clone 再上传
      git clone https://github.com/qiongtu2077/dongxuelian-qqbot.git
      scp dongxuelian-qqbot/setup.sh root@服务器IP:/root/

2. ssh 登录服务器

   ssh root@服务器IP -p 端口

3. 编辑 setup.sh 顶部的两个参数

   QQ_NUMBER="你的机器人QQ号"
   ADMIN_QQ="你的QQ号（管理员）"

4. 运行

   bash setup.sh

   脚本会自动完成以下操作：
   - 安装系统依赖（Node.js 18+、git、tesseract-ocr、ffmpeg、xvfb 等）
   - 运行 NapCat 官方安装器（安装 LinuxQQ + NapCat + CLI 工具）
   - 写入 NapCat 配置（onebot WS 端口 8080）
   - 从 GitHub 克隆本仓库到 /root/koishi-app
   - 执行 npm install
   - 创建数据目录和默认配置文件
   - 启动 NapCat → 显示二维码

5. 看到二维码后，掏出手机 QQ 扫码登录

6. 登录成功后，按提示输入 API key
   - 支持的供应商 key 均可选；没有 key 时可以直接回车跳过
   - 跳过不会阻塞基础部署，后续可在 Dashboard 的 API Keys 页补充

7. 自动启动 Koishi，部署完成

验证：tail -f /root/koishi-app/koishi.log
看到 Dashboard、Koishi 和主要插件 loaded 日志，并且 `AI状态` 能返回，即基础部署成功。


方式二：手动部署（每步可控）
------------------------------

1. 安装 Node.js 18+

   curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
   apt-get install -y nodejs

2. 安装系统依赖

   apt-get update
   apt-get install -y curl wget git unzip jq xvfb screen procps \
     tesseract-ocr tesseract-ocr-chi-sim ffmpeg python3 python3-pip
   pip3 install yt-dlp

3. 克隆仓库并安装依赖

   git clone https://github.com/qiongtu2077/dongxuelian-qqbot.git /root/koishi-app
   cd /root/koishi-app
   npm install

4. 安装 NapCat（官方安装器）

   curl -k -L -# -o napcat.sh "https://nclatest.znin.net/NapNeko/NapCat-Installer/raw/main/install.sh"
   bash napcat.sh --docker n --cli y --proxy 0 --force

5. 写入 NapCat 配置

   NAPCAT_DIR="/root/Napcat/opt/QQ/resources/app/app_launcher/napcat/config"
   mkdir -p "$NAPCAT_DIR"

   # 主配置
   cat > "$NAPCAT_DIR/napcat.json" <<'EOF'
   {
     "fileLog": false,
     "consoleLog": true,
     "fileLogLevel": "debug",
     "consoleLogLevel": "info",
     "packetBackend": "auto",
     "packetServer": "",
     "o3HookMode": 1,
     "bypass": { "hook": false, "window": false, "module": false,
       "process": false, "container": false, "js": false },
     "webui": { "port": 6099, "token": "123" }
   }
   EOF

   # onebot 配置（将 你的QQ号 替换为实际号码）
   cat > "$NAPCAT_DIR/onebot11_你的QQ号.json" <<EOF
   {
     "network": {
       "httpServers": [],
       "httpSseServers": [],
       "httpClients": [],
       "websocketServers": [
         { "name": "WebSocket Server", "enable": true,
           "host": "127.0.0.1", "port": 8080 }
       ],
       "websocketClients": [],
       "plugins": []
     },
     "musicSignUrl": "",
     "enableLocalFile2Url": false,
     "parseMultMsg": false,
     "imageDownloadProxy": "",
     "timeout": {
       "baseTimeout": 10000,
       "uploadSpeedKBps": 256,
       "downloadSpeedKBps": 256,
       "maxTimeout": 1800000
     }
   }
   EOF

6. 创建数据目录和初始配置（在 /root/koishi-app 目录下）

   DATA_DIR="/root/koishi-app/data"
   mkdir -p "$DATA_DIR/ai-skills/core" "$DATA_DIR/ai-skills/personas" \
     "$DATA_DIR/ai-skills/modes" "$DATA_DIR/ai-skills/lore" \
     "$DATA_DIR/user-profiles" "$DATA_DIR/conversations" \
     "$DATA_DIR/ai-event-dumps" "$DATA_DIR/political-handlers"

   # 复制 SKILL 文件
   cp -rn packages/koishi-plugin-dongxuelian-ai/data/ai-skills/core/* \
     "$DATA_DIR/ai-skills/core/" 2>/dev/null || true
   cp -rn packages/koishi-plugin-dongxuelian-ai/data/ai-skills/personas/* \
     "$DATA_DIR/ai-skills/personas/" 2>/dev/null || true

   # 默认配置
   echo "opencode" > "$DATA_DIR/ai-provider.txt"
   echo "deepseek-v4-flash" > "$DATA_DIR/ai-model.txt"
   echo "https://opencode.ai/zen/go/v1" > "$DATA_DIR/ai-base-url.txt"
   echo "[]" > "$DATA_DIR/ai-random-whitelist.json"
   echo "[]" > "$DATA_DIR/ai-user-blacklist.json"
   echo "{}" > "$DATA_DIR/ai-repeat-enabled.json"
   echo "off" > "$DATA_DIR/ai-enable-search.txt"
   echo "off" > "$DATA_DIR/ai-enable-thinking.txt"

7. 写入 koishi.yml（在 /root/koishi-app 目录下）

   cat > /root/koishi-app/koishi.yml <<EOF
   plugins:
     server:emicam:
       port: 5140
       selfUrl: http://localhost:5140
     adapter-onebot:xtqqgv:
       protocol: ws
       selfId: '你的QQ号'
       endpoint: ws://127.0.0.1:8080/onebot/v11/ws
     group-name-at:nyxxfd: {}
     dongxuelian-help:rlmpxx: {}
     dongxuelian-ai:hdi04m: {}
     dongxuelian-poke:nxf8l0: {}
     koishi-plugin-defense:xlyp9f: {}
     local-video-sender:k2w0u7: {}
     group-leave-notice:h6lfrz: {}
   EOF

8. 启动 NapCat 并扫码

   screen -dmS napcat bash -c \
     "xvfb-run -a /root/Napcat/opt/QQ/qq --no-sandbox -q 你的QQ号"
   screen -r napcat

   看到二维码后扫码登录，然后 Ctrl+A → D 分离 screen

9. 写入 API key（可选）

   echo "sk-你的key" > /root/koishi-app/data/ai-deepseek-key.txt
   echo "deepseek" > /root/koishi-app/data/ai-provider.txt
   echo "deepseek-chat" > /root/koishi-app/data/ai-model.txt
   echo "https://api.deepseek.com" > /root/koishi-app/data/ai-base-url.txt

   也可以换成其他供应商；如果暂时没有 key，可先跳过这一步，基础部署仍可继续。
   其他 key（按需）：
   echo "tp-xxx" > /root/koishi-app/data/ai-mimorium-key.txt
   echo "sk-xxx" > /root/koishi-app/data/ai-openai-key.txt
   echo "sk-xxx" > /root/koishi-app/data/ai-dashscope-key.txt

10. 启动 Koishi

    cd /root/koishi-app
    nohup npx koishi start >> koishi.log 2>&1 &

11. 验证

    tail -f /root/koishi-app/koishi.log
    看到 Dashboard、Koishi 和主要插件 loaded 日志，并且 `AI状态` 能返回，即基础部署成功


更新部署（已有服务器，只更新代码）
------------------------------------

方式 A：全量更新（精简版）
  从本地仓库根目录：
  scp packages/koishi-plugin-dongxuelian-ai/lib/*.js \
    root@服务器IP:/root/koishi-app/packages/koishi-plugin-dongxuelian-ai/lib/
  scp packages/koishi-plugin-dongxuelian-help/lib/index.js \
    root@服务器IP:/root/koishi-app/packages/koishi-plugin-dongxuelian-help/lib/
  scp packages/koishi-plugin-group-name-at/lib/index.js \
    root@服务器IP:/root/koishi-app/packages/koishi-plugin-group-name-at/lib/
  scp packages/koishi-plugin-local-video-sender/lib/index.js \
    root@服务器IP:/root/koishi-app/packages/koishi-plugin-local-video-sender/lib/
  scp packages/koishi-plugin-dongxuelian-ai/data/ai-skills/core/* \
    root@服务器IP:/root/koishi-app/data/ai-skills/core/
  scp "packages/koishi-plugin-dongxuelian-ai/data/ai-skills/personas/SKILL.长离.md" \
    root@服务器IP:/root/koishi-app/data/ai-skills/personas/
  scp "packages/koishi-plugin-dongxuelian-ai/data/ai-skills/personas/SKILL.椿.md" \
    root@服务器IP:/root/koishi-app/data/ai-skills/personas/
  scp "packages/koishi-plugin-dongxuelian-ai/data/ai-skills/personas/SKILL.特蕾西娅.md" \
    root@服务器IP:/root/koishi-app/data/ai-skills/personas/
  # 重启 Koishi
  ssh root@服务器IP "pkill -f 'koishi/lib/worker'; sleep 3; cd /root/koishi-app && npx koishi start"

方式 B：scripts/ 按插件更新（推荐）
  scripts/ 目录下每个插件都有独立的部署脚本，共享 deploy-package.sh 工具：

  scripts/
  ├── deploy-package.sh     ← 共享部署工具（语法检查、koishi.yml 注册、安全路径校验）
  ├── ai.sh                 ← 部署 AI 插件（含 SKILL 文件）
  ├── help.sh               ← 部署 help 插件
  ├── name.sh               ← 部署 group-name-at 插件
  ├── poke.sh               ← 部署 poke 插件
  ├── defense.sh            ← 部署 defense 插件
  ├── leave.sh              ← 部署 group-leave-notice 插件
  ├── vedio.sh              ← 部署 local-video-sender 插件
  └── message-reader.sh     ← 部署 message-reader.js

  在本地仓库根目录执行：

  单个插件更新：
  bash scripts/ai.sh          # 只更新 AI 插件（含语法检查 + koishi.yml 注册）
  bash scripts/help.sh        # 只更新 help 插件
  bash scripts/name.sh        # 只更新群昵称插件

  全部更新：
  for f in scripts/*.sh; do bash "$f"; done

  每个脚本会自动：
  - 从 packages/ 复制文件到服务器的 node_modules 对应目录
  - 对每个 .js 文件执行 node -c 语法检查
  - 在 koishi.yml 中注册插件（如果尚未注册）
  - 支持设置部署目标目录（通过 KOISHI_APP_DIR 环境变量）

方式 C：远程一键重启（已经部署完成后快速重启）
  如果你只是修改了 koishi.yml 或 data 目录下的配置文件，不需要更新代码：

  ssh root@服务器IP "cd /root/koishi-app && pkill -f 'koishi/lib/worker' && sleep 3 && npx koishi start"


环境变量覆盖
--------------

部署时可设置环境变量覆盖默认路径，避免依赖服务器特定路径：

  DONGXUELIAN_DATA_DIR     → data 目录（默认 /root/koishi-app/data）
  GROUP_NAME_AT_DATA_FILE  → 群昵称数据文件（默认 $DATA_DIR/nickname-collections.json）
  BILI_COOKIES_FILE        → B站 cookies（默认 /root/bilibili-cookies.txt）
  BILI_WORKDIR             → 视频下载目录（默认 /root/koishi-bili-downloads）
  BILI_MAX_SIZE_BYTES      → 视频大小限制（默认 200MB）
  BILI_TEST_VIDEO_FILE     → 测试视频路径（默认 /root/test_bili.mp4）
  KOISHI_APP_DIR           → Koishi 部署目录（默认 /root/koishi-app，scripts/ 部署用）

  例：
  DONGXUELIAN_DATA_DIR=/custom/data bash scripts/ai.sh


常用命令
--------

查看 bot 日志：              tail -f /root/koishi-app/koishi.log
重启 Koishi：               pkill -f 'koishi/lib/worker'; sleep 3; cd /root/koishi-app && npx koishi start
查看 NapCat（看二维码）：    screen -r napcat（Ctrl+A D 分离）
重启 NapCat：               screen -S napcat -X quit
                             screen -dmS napcat bash -c \
                               "xvfb-run -a /root/Napcat/opt/QQ/qq --no-sandbox -q 你的QQ号"
运行测试：                  cd /root/koishi-app && npm test
仅跑插件测试：               cd /root/koishi-app && npm run test:plugins


注意事项
--------

1. 杀进程用 pkill -f 'koishi/lib/worker'
   不要用 pkill -f "start"（会误杀 SSH 守护进程，导致服务器失联）

2. SCP 传输文件时直接用二进制传
   不要用 PowerShell 的 Get-Content 管道（会转 UTF-16 损坏中文和 API key）

3. API key 格式
   DeepSeek  → sk-xxx
   mimorium  → tp-xxx
   opencode  → sk-xxx
   dashscope → sk-xxx
   GLM       → 无固定前缀（原样粘贴）

4. 更新代码后，如果 package.json 新增了依赖，服务器上要重新 npm install

5. 如果修改了 koishi.yml，需要重启 Koishi 才能生效

6. NapCat 首次启动必须扫码登录，这是唯一需要人工操作的步骤

7. scripts/ 部署脚本会在部署前自动对每个 .js 文件执行 node -c 语法检查，
   检查不通过时部署会停止，不会损坏已运行的 bot
