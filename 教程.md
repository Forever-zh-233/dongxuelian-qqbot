# Bot 接入 AI + Skill 简明教程

这份教程只讲你现在这套 Koishi 机器人的落地思路。

## 一、整体结构

你现在的方案可以拆成 4 层：

- `Koishi`：机器人框架。
- `dongxuelian-ai`：你自己的 AI 聊天插件。
- `OpenAI 兼容接口`：例如 Qwen 的兼容模式接口。
- `skill`：放在服务器上的补充设定或风格文件。

关系是：

```text
QQ群消息 -> Koishi -> dongxuelian-ai -> 模型接口 -> 返回回复
                                 \
                                  -> 读取本地 skill
```

## 二、AI 插件负责什么

AI 插件主要做 5 件事：

- 判断这条消息要不要触发 AI。
- 过滤掉昵称、集合、帮助等保留指令。
- 读取系统设定和 skill。
- 调用模型接口。
- 把模型回复发回群里。

## 三、Skill 负责什么

skill 不是模型，也不是训练。

它更像是：

- 一份长期提示词
- 一套风格说明
- 一组行为约束

例如你可以把这些内容写进 skill：

- 角色设定
- 说话风格
- 禁止事项
- 你自己的思考习惯
- 某类任务的固定处理方式

## 四、用 skill“蒸馏”自己的聊天是什么意思

如果你说的“蒸馏”是想让 AI 更像你，一般不是直接训练模型，而是分两步：

### 第一步：整理你的聊天风格

把你平时的表达抽象成规则，例如：

- 你常用什么句式
- 你如何吐槽
- 你怎么解释问题
- 你会不会先说结论
- 你是否喜欢简短回复

这些内容写成 `SKILL.md`。

### 第二步：让插件每次都带上 skill

这样模型每次回答前，都会先读这份风格说明。

结果就是：

- 不需要真训练
- 成本低
- 可随时改
- 很适合你现在的私有 bot

## 五、什么时候才需要真训练

只有在下面这些情况下，才考虑微调或更重的方案：

- 你有大量高质量样本。
- 你希望模型稳定模仿某种固定表达。
- 单靠 prompt / skill 已经不够。

对你现在的 bot 来说，优先级应该是：

1. 先把插件跑通
2. 再把 skill 调顺
3. 最后才考虑训练

## 六、推荐落地顺序

### 1. 先部署 AI 插件

先确认：

- `ai-openai-key.txt`
- `ai-model.txt`
- `ai-base-url.txt`

都已放在服务器正确位置。

### 2. 再把 skill 上传到服务器

默认目录：

```text
/root/koishi-app/data/ai-skills
```

### 3. 发送 `AI重载`

让插件重新读取 skill 和配置。

### 4. 最后测试两类触发

- 普通群聊随机触发
- `@机器人` 必定触发

## 七、你现在最该做的

如果你要的是“先能用”，最现实的路线是：

- 先保证 `ai.md` 部署脚本正确
- 再上传你的 skill
- 再用几轮真实聊天慢慢调风格

不要一开始就想着训练模型本体。

对你当前阶段，`skill + 系统提示词 + 少量规则约束` 就够用了。
