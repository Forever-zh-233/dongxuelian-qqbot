# Bot 接入 AI 与 Skill 蒸馏教程

更新时间：2026-04-20

## 这篇教程讲什么

这篇教程只讲三件事：

1. 怎么给你现在这个 Koishi Bot 接一个 AI。
2. 怎么让这个 AI 带着一个 `skill` 说话。
3. 怎么把你自己的聊天风格“蒸馏”成一个可复用的 `skill`。

这里的“蒸馏”不是训练大模型参数，而是把你的聊天记录整理成：

- 人设
- 口头禅
- 回复风格
- 禁忌
- 常见问答模板
- few-shot 示例

然后在 AI 回复时一起喂给模型，让它更像你。

---

## 一、你现在的 Bot 结构

你现在这套是：

```text
QQ / NapCat -> OneBot11 -> Koishi -> 你的自定义插件
```

如果要接 AI，本质上就是再加一层：

```text
QQ消息 -> Koishi插件 -> 调 OpenAI API -> 拿到回复 -> 发回群里
```

所以最自然的做法是：

- 新建一个独立插件，比如 `ai-chat.md`
- 这个插件只负责 AI 对话
- 现有的：
  - `name.md`
  - `leave.md`
  - `vedio.md`
  - `help.md`
  都不要混进去

这样最稳。

---

## 二、什么叫“给 Bot 接一个 AI”

就是在 Koishi 里写一个插件，监听消息，然后：

1. 判断这条消息是不是要交给 AI。
2. 如果是，就把消息发给模型。
3. 把模型返回内容回复到群里。

最常见的触发方式有三种：

### 方案 A：命令触发

比如：

```text
AI 你好
```

优点：

- 最安全
- 不会乱接话
- 好调试

缺点：

- 不够像“常驻群聊机器人”

### 方案 B：前缀触发

比如：

```text
东雪莲 你好
```

优点：

- 更自然
- 也比较容易控范围

缺点：

- 需要处理前缀裁剪

### 方案 C：@机器人 触发

比如：

```text
@东雪莲 今天吃什么
```

优点：

- 群聊体验最好
- 不容易误触发

缺点：

- 要处理 at 解析

如果你是第一次接 AI，我建议先做：

```text
AI 你的问题
```

先跑通，再升级到 `@机器人`。

## 三、什么叫“AI 使用一个 skill”

最简单理解：

`skill` 就是一份额外提示词说明书。

它不是模型权重，不是微调文件，而是一份结构化规则，告诉模型：

- 你是谁
- 你怎么说话
- 你擅长什么
- 你回答时遵守什么格式
- 你不能做什么

比如你可以有一个 `dongxuelian.skill.md`：

```md
# 角色
你是东雪莲风格群聊助手。

# 说话风格
- 说话自然
- 不端着
- 偶尔有吐槽感
- 句子不要太长

# 禁止
- 不要像客服
- 不要每句都很正式
- 不要长篇大论

# 群聊规则
- 优先短回复
- 用户问得短，你就短
- 被骂不要破防，轻微回怼但别失控
```

然后每次调用模型时，把这份内容放进 system prompt 或 developer prompt。

这样模型就“带着 skill” 回答了。

---

## 四、Skill 和普通提示词的区别

普通提示词通常是一小段：

```text
你现在是一个可爱的助手。
```

而 skill 更像一份长期维护的配置：

- 可以单独存文件
- 可以不断迭代
- 可以按功能拆分
- 可以做版本管理

比如你可以拆成：

```text
skills/
  core-persona.md
  group-style.md
  roast-style.md
  safety.md
  reply-format.md
```

然后在插件里把这些 skill 拼起来再发给模型。

## 五、你说的“蒸馏我自己的聊天”到底是什么

你要的其实是：

**从自己的聊天记录里提炼出稳定风格，并固化成 skill。**

这件事最推荐分 3 层做。

### 第一层：提炼规则

从你的聊天记录里总结：

- 你常用什么称呼
- 你一句话一般多长
- 你会不会用脏字
- 你阴阳怪气的强度
- 你喜欢问句还是陈述句
- 你会不会补刀
- 你什么时候认真，什么时候敷衍

这层产物是：

```text
风格规则
```

### 第二层：提炼示例

从你的真实聊天里整理成 few-shot：

```text
用户：今天吃啥
你：吃土最省

用户：你会不会说人话
你：会，但今天不太想会
```

这层产物是：

```text
对话样例
```

### 第三层：提炼边界

把你不想让 AI 学到的东西排出去：

- 情绪失控
- 真正会伤人的内容
- 隐私
- 涉黄涉暴过线内容
- 你只是偶尔说过，但不代表长期风格的句子

这层产物是：

```text
禁止规则
```

## 六、最推荐的蒸馏流程

你可以按这个流程做。

### 第 1 步：收集聊天记录

先把你自己的聊天记录整理成文本。

格式尽量统一，例如：

```text
用户：你今天怎么这么安静
我：懒得理世界

用户：出来玩
我：没空，忙着当废物
```

不要一上来就全量喂几万条。

建议先选：

- 100 条你最像自己的回复
- 50 条典型嘴臭/吐槽回复
- 50 条认真解释型回复

总共 200 条左右先做第一版。

### 第 2 步：人工清洗

把这些内容分成三类：

#### 保留

- 最像你风格的
- 可重复复用的
- 不涉及隐私的

#### 改写后保留

- 风格像你，但内容太私人
- 风格像你，但太攻击

#### 删除

- 情绪崩溃时说的话
- 纯发泄
- 明显不适合机器人学的内容

### 第 3 步：总结成 skill 结构

建议整理成下面这个模板：

```md
# 角色定位

# 说话风格

# 常用语气

# 常用句式

# 回答长度偏好

# 什么时候简短

# 什么时候认真解释

# 禁止事项

# few-shot 示例
```

### 第 4 步：先走 prompt 蒸馏，不要先训练

很多人一上来就想“训练一个自己的模型”。

对你现在这个 Bot 场景，我不建议先这么做。

先用：

- system prompt
- skill 文件
- few-shot 示例

这三样，往往已经能做出 70% 到 90% 的风格效果。

先跑起来，再决定要不要做更重的训练。

## 七、实际落地时怎么接到 Bot 里

你最终需要一个 AI 插件，大概做这几步：

### 1. 读取 skill 文件

例如：

```text
/root/koishi-app/data/ai-skills/dongxuelian.md
```

### 2. 拼系统提示词

比如：

```text
系统提示 = 基础安全规则 + 角色设定 + 你的 skill + few-shot
```

### 3. 拿用户消息一起发给模型

结构上大概像：

```json
[
  { "role": "system", "content": "基础规则..." },
  { "role": "system", "content": "skill内容..." },
  { "role": "user", "content": "用户这次说的话" }
]
```

### 4. 拿回复发回 QQ

就这么简单。

## 八、推荐的 Skill 文件结构

你可以以后这样放：

```text
data/
  ai/
    skills/
      dongxuelian-core.md
      dongxuelian-style.md
      dongxuelian-examples.md
      dongxuelian-safety.md
```

然后在 AI 插件里合并：

```text
core + style + examples + safety
```

这比把所有东西写一坨更容易维护。

## 九、Skill 蒸馏模板

你可以直接照这个格式做第一版：

```md
# 角色
你是一个群聊型 AI 助手，风格参考“我本人”。

# 风格
- 回答短一点
- 偶尔吐槽
- 不装客服
- 不要太书面

# 语气
- 自然
- 有点损
- 但不过线

# 回复长度
- 日常闲聊 1~2 句
- 解释问题时 3~5 句

# 常见表达
- 行
- 不至于
- 你这就离谱了
- 有点东西

# 禁止
- 不要每次都说很多
- 不要像官方公告
- 不要无脑复读用户原话
- 不要过度攻击群友

# Few-shot
用户：你在吗
助手：活着

用户：你怎么这么会说
助手：瞎说的

用户：教我一下
助手：先把问题说人话
```

## 十、如果你真的想“训练”

如果后面你不是只想做 prompt/skill，而是真的想做更深的风格固化，一般有三条路：

### 路线 A：只做 Prompt + Skill

最轻，最适合你现在。

优点：

- 快
- 成本低
- 好改

缺点：

- 稳定性不如训练

### 路线 B：做 RAG / 示例召回

把你的历史聊天做成一个小知识库。

每次用户发问时：

1. 先从你的历史聊天里检索最像的几条
2. 再把这些示例一起喂给模型

优点：

- 风格拟合明显加强
- 不需要训练模型

缺点：

- 实现比纯 prompt 复杂一点

### 路线 C：做微调 / 蒸馏训练

这是最重的一条。

只有在下面这些情况才建议考虑：

- 你已经有大量高质量对话数据
- 你明确知道自己要训练什么风格
- 你能接受更高维护成本

对你现在这套 QQ Bot，我建议顺序是：

```text
先 Prompt/Skill -> 再 Few-shot -> 再 RAG -> 最后才考虑训练
```

## 十一、最实用的落地建议

如果你现在就想做，最建议的路线是：

### 第一步

先新建一个独立插件：

```text
ai.md
```

只做：

- `AI 你的问题`
- 调模型
- 发回复

### 第二步

再新建一个 skill 文件：

```text
东雪莲人格.md
```

写你的人设、风格、禁忌、few-shot。

### 第三步

把这个 skill 在每次请求时读进去。

### 第四步

用你自己的聊天记录，逐步迭代这个 skill。

不要一开始就搞训练。

如果你要我帮你继续落地，我建议顺序是：

2. 再给你写一个 `东雪莲人格.md`

---

## 十三、一句话总结

你要的“skill 蒸馏自己的聊天”，本质上就是：

```text
把你的聊天记录 -> 清洗 -> 提炼风格规则和样例 -> 写成 skill 文件 -> 在 Bot 调 AI 时一起喂给模型
```

先别急着训练模型，先把 `skill + few-shot` 跑起来，效果通常已经很够用了。
