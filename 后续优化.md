## 一、Agent QQ 回复优化

### 当前问题

| # | 问题 | 根因 |
|---|------|------|
| A | Agent 回复跳过人格 | `index.js:1157` Agent 结果直接 `safeSendReply`，不走 `chat()` |
| B | Agent 回复无长度截断 | 同上。QQ 单条上限 ~4500 字，Agent 搜索结果经常超限 |
| C | 搜索 query 泄漏到 QQ 回复 | `chat.js:857` followUp 透传了模型生成的 query 文本 |
| D | 模型用 2025 搜不到最新结果 | `chat.js:480` 时间注入只有时分，缺年月日 |

### 目标架构

```
QQ 消息
  │
  ├─ 普通消息 → chat() → safeSendReply()           (不变)
  │
  └─ 需 Agent 消息 → agentEngine.run({ agentMode: true })
       │   Agent 返回纯直连结果（无人格、无 OOC 守卫）
       │
       └─→ chat({ isAgentResult: true, agentResult })
           │   chat 拿到 Agent 结果注入为上下文
           │   chat 用当前角色口吻自然转述
           │
           └─→ safeSendReply()                     (自动截断/人格/检测)
```

### 效果对比

| | 现在 | 改为 |
|---|---|---|
| QQ 用户看到什么 | Agent 裸结果 | 角色在说话，感知不到 Agent |
| 长度控制 | 可能超限 | chat 已自带截断 |
| 人格一致性 | AgentMode toggle 控制 | 永远直连 + chat 补人格 |
| query 泄漏 | chat followUp 可能泄漏 | 固定 '让我看看…' |
| 年份 | 模型不知 | 注入年月日 |
| Dashboard 前端 | 完整 reply + rounds | 完整 reply + rounds，不受影响 |

### 改动清单

| # | 文件 | 位置 | 改动 | 行数 |
|---|------|------|------|:---:|
| 1 | `index.js` | line 1154 | Agent QQ 渠道强制 `agentMode: true` | +1 |

| 3 | `chat.js` | 新增函数/分支 | `options.isAgentResult` 时注入 Agent 结果作为上下文，模型用角色转述 | +15 |

| 5 | `chat.js` | line 480 | 时间注入加年月日：`${year}年${month}月${day}日 ${hour}时${min}分` | ±1 |

**总计**：3 个文件，~30 行净增。

## 五、图片历史 + MCP 接入 + 图片工具（本次讨论内容）

### 5.1 vision.js 接入 image-store

**文件**：`lib/vision.js`

**问题**：`appendVisionMessage` 实时识图成功后，结果没有写回 image-store 和 conversation 文件。分析结果只活在当前 API 调用，下一轮就读不到。

**改动**：在 `finally` 块中的 `clearVisionSession(session)` 之前，调用 `markAnalyzed()` 标记图片已分析，后续 `analyze_historical_image` 不会再重复下载。

大约 10 行。

---

### 5.2 chat-tools.js 加图片工具

**文件**：`lib/chat-tools.js`

**问题**：Feature 2（Chat 轻量 Agent）已上线，正常聊天和随机回复中 LLM 有工具权限，但 `read_image_history` 和 `analyze_historical_image` 不在 chat-tools.js 里。LLM 看到 `[图片]` 占位符时没有工具可查，只能瞎编。

**改动**：
- `LIGHTWEIGHT_TOOLS` 加 `read_image_history`、`analyze_historical_image`
- `getChatToolDefinitions()` 追加两个 tool definition
- `executeChatTool()` 追加两个 case

大约 20 行。

### 5.3 图片缓存条目数调整

**文件**：`lib/image-store.js`

**问题**：`MAX_IMAGES_PER_CHANNEL = 20` 偏大。每条包含 URL 和分析结果，单频道的缓存窗口控制得更紧。

**改动**：`20 → 10`

1 行。URL 2 小时过期 + 每频道 10 条上限。conversation 里的 `[图片]: xxx` 不受影响，永久保留。

### 5.4 MCP 浏览器接入 web-search 搜索链

**文件**：`lib/agent/tools/web-search.js`

**问题**：MCP Server 框架已建好（`lib/mcp/client.js` + `packages/mcp-browser-server/`），但 `web-search.js` 的 fallback 链没接 MCP。

**改动**：API 搜索失败后，加一档 MCP `browser_search`。在线时优先用远程浏览器（能渲染 SPA），不在线时降级到 HTTP 搜索。
