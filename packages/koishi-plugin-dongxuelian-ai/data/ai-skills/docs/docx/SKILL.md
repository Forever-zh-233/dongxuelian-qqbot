# DOCX Skill

用户要求生成或处理 Word、DOCX、报告、备忘录、合同草稿、模板、信函、简历、表格文档时使用。

## 当前原则

- `.docx` 是 zip 包中的 XML。简单生成可用 JS 库，精细编辑通常需要解包后修改 XML。
- 不覆盖原文件，除非用户明确要求。默认输出新文件。
- 格式比纯文本更脆弱，编辑后必须验证文件能打开。

## 创建 DOCX

1. 确认文档用途、语言、页眉页脚、目录、标题层级、表格、图片和页码要求。
2. 结构化内容：封面、摘要、章节、表格、附录。
3. 使用可用依赖生成 DOCX；缺少依赖时说明，不要伪造文件。
4. 标题使用真实 heading 样式，目录依赖 heading 层级。
5. 列表使用文档库的编号/项目符号功能，不手写特殊符号冒充列表。
6. 表格设置明确宽度、边框和单元格内边距。

## 读取与编辑 DOCX

1. 需要理解内容时，优先用 LibreOffice、pandoc 或解包 XML 抽取文本。
2. 需要局部替换时，先定位 `word/document.xml` 中的目标文本；注意一个词可能被拆成多个 run。
3. 涉及批注/修订时，保持 XML 结构合法，不把修订标签塞进 run 内部。
4. 图片替换需要同步 `word/media/`、`word/_rels/document.xml.rels` 和 `[Content_Types].xml`。
5. 完成后重新打包，并用 LibreOffice 或解包检查验证。

## 安全边界

- 不伪造公章、签名、证明文件或法律结论。
- 不把密钥、cookie、群聊隐私写进文档。
- 法律、财务、医疗文档只做格式整理和文字归纳，必要时提醒用户找专业人士确认。

## 渠道能力

- QQ Agent 默认不能写文件。若用户在 QQ 要求生成 DOCX，应说明需要 Dashboard Agent 或管理员开启并审批写入/执行工具。
- Dashboard Agent 可使用文件工具、shell、浏览器和发送文件能力完成实际交付。
