# PPTX Skill

用户明确要求制作“PPT、PPTX、演示文稿、幻灯片、汇报材料”时使用。先确认交付目标、受众、页数、语言、风格和是否需要真实数据来源。

## 当前项目能力

- 本项目已有生成入口：`pptx/generate-pptx.js`
- 旧版兼容说明：`pptx/SKILL.pptx-generator.md`
- Dashboard Agent 可在审批后使用文件写入、shell、浏览器截图等工具。
- QQ Agent 默认没有文件写入、shell、浏览器权限；QQ 中需要生成真实文件时，应说明需要切到 Dashboard 或由管理员开启/审批相关工具。

## 创建 PPTX

1. 把需求整理为结构化大纲：标题、页数、每页标题、bullet、备注、素材来源。
2. 若信息需要更新，先用 `web_search` 或浏览器工具查可靠来源，并在备注中保留来源摘要。
3. 生成 slides JSON，每页最多 7 条 bullet，标题尽量不超过 40 字。
4. 将输出写到允许工作区，例如 `data/agent-docs/` 或用户指定且已授权的目录。
5. 使用项目脚本或现有依赖生成 PPTX。缺少依赖时明确报告，不要伪造“已生成”。
6. 生成后检查文件存在、大小合理，并尽可能用 LibreOffice、浏览器截图或解包 XML 做快速验证。

## 编辑 PPTX

1. 先备份原文件到同目录或临时输出名。
2. 读取现有结构，可用解包、LibreOffice 转换、文本抽取或截图辅助理解。
3. 只改用户要求的页和元素，保留原主题、母版和版式。
4. 完成后检查文件能打开，必要时导出 PDF 或图片进行视觉核验。

## 设计约束

- 不把 API Key、cookie、私密群聊原文写进 PPT。
- 不把未经验证的联网信息当事实。
- 幻灯片应直接服务任务，避免营销式空话。
- 如果用户要求“专业汇报”，优先选择清晰层级、少量颜色、可扫读表格和明确结论。
